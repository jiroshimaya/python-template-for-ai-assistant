# ADR 0001: Architecture Decision Records を記録する

## ステータス

採用

## コンテキスト

このリポジトリでは、開発フローや設計判断を継続的に残していきたいが、判断理由を追跡できる置き場がまだなかった。

判断の背景がコミットメッセージや pull request だけに分散すると、あとから「なぜその方針を選んだのか」を把握しづらい。

## 決定

`docs/adr/` 配下に Architecture Decision Record（ADR）を連番で保存する。

ADR は以下の方針で管理する。

- ファイル名は `NNNN-short-title.md` 形式にする
- 文書タイトルも同じ番号を持つ
- 後続の判断は次の連番で追加する

## 影響

- 重要な判断とその理由を、リポジトリ内で継続的に参照できる
- 今回以降の設計・運用判断は ADR として積み上げやすくなる
